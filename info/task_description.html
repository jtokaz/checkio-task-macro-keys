A keyboard macro allows a sequence of keystrokes to be transformed into another usually longer sequence of keystrokes.

For this task someone has maliciously assigned macros to single letter keystrokes. If d is assigned to foo and s is assigned to bar then typing "I like dessert" results in the screen displaying "I like fooebarbarert". You know what you typed and what you seen displayed on the screen. Use your coding skills to determine the minimum number of letters that were assigned macros.

Only letters are assigned macros, and these can be any length including zero. The upper and lower cases of a letter are mapped separately. The letter A could be mapped to b, while a is mapped to bar or nothing at all.

Input: The string you typed and the string displayed with macros applied.

Output: The minimal number of macros required to transform the typed string to the displayed string.

Example: 

checkio("I like dessert","I like fooebarbarert")
    # should return 2

checkio([
    [0, 0, 0],
    [0, 5, 0],
    [0, 0, 0]
])
#can return [[2, 7, 6], [9, 5, 1], [4, 3, 8]] or
# [[4, 9, 2], [3, 5, 7], [8, 1, 6]

checkio([[1, 15, 14, 4],
         [12, 0, 0, 9],
         [8, 0, 0, 5],
         [13, 3, 2, 16]])
# answer [[1, 15, 14, 4], [12, 6, 7, 9], [8, 10, 11, 5], [13, 3, 2, 16]]

How it is used: This is a constraint satisfaction problem. It's used not only for solving puzzles, but also for planning and resource allocation in city planning, construction and just about everywhere else.
